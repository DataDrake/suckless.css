Colors

Motivation
As a designer the use of color can be used to transform a boring black and white canvas into something extraordinary.
Philosophy
Colors should be applied at whatever scope they are needed. Rules for colors should never involve anything that is unrelated to color.

Description

Three custom color classes are provided by default. They may serve as the basis for a color palette and their specific values are set according to the imported theme. More colors may be defined using the included mixin.

.warn
.warn-i
Error
.error
.error-i
Description

Four status color classes are provided by default. These classes may be used as visual indicators that a particular status is being expressed by the contents.

Info
The contents indicate some relevant information that a viewer might find useful, but is neutral in severity.
Good
The contents indicate that something positive is being expressed, ie. an operation completed successfully.
Warn
The contents indicate that a potential concern is being expressed and that the viewer should take note.
Error
The contents indicate the presence of a failure or the potential for future failure.

Mixins

@mixin color($name,$fg,$bg)
Create a new color class and it's inverted counterpart
$name
Name of the class
$fg
Foreground color for normal, Background color for inverse
$bg
Background color for normal, Foreground color for inverse
